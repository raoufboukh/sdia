💡 What Else Could Be Added to This Project
Extended ideas for a more complete bioinformatics pipeline
1. Gene Name Annotation
After selecting top 94 genes by ANOVA/MI, map their indices back to real gene names using the dataset's column headers. Then use KEGG or Gene Ontology (GO) to find which biological pathways they belong to.
2. t-SNE / UMAP Visualization
Reduce 3,571 genes to 2D using t-SNE or UMAP (better than PCA for visualization). Plot each patient as a dot — well-separated clusters = good class structure. Shows data quality visually.
3. Nested Cross-Validation
Use outer loop for evaluation, inner loop for hyperparameter tuning (GridSearchCV). Prevents optimistic bias from tuning on the same data used to evaluate. Gold standard for small medical datasets.
4. SMOTE — Synthetic Oversampling
Generate synthetic tumor samples to balance the 47/25 split before training. Can improve recall (sensitivity) for the minority class. Must be applied inside CV folds only.
5. Ensemble Voting Classifier
Combine SVM + RF + KNN predictions by majority vote. Often more robust than any single model. Soft voting (probability average) is even better when classifiers are well-calibrated.
6. Learning Curves
Plot accuracy vs number of training samples. Shows whether the model suffers from high bias (underfitting) or high variance (overfitting). Critical for a 72-patient dataset.
Contents

🧬 Project Overview

This project is a bioinformatics machine learning study that analyzes gene expression data to classify cancer patients. Gene expression microarrays measure how active each of thousands of genes is in a tissue sample — producing one number per gene per patient.

Two datasets are used:

  • PONE dataset (augmented): 94 patients, 3,571 genes — labels: Normal vs Tumar (47/47)
  • Golub dataset: 72 patients, 7,129 genes — labels: ALL leukemia vs AML leukemia

The main challenge is that you have 3,571 features but only 94 samples — a classic "p >> n" problem in bioinformatics. This makes dimensionality reduction and feature selection essential.

Why gene expression? Different cancer types have different gene activity signatures. A tumor cell activates oncogenes and silences tumor suppressors differently than a normal cell. ML can detect these patterns automatically across thousands of genes simultaneously.

StandardScaler is critical: it transforms each gene to have mean=0 and std=1. Without this, genes with large numerical values (e.g., expression = 10,000) dominate over genes with small values (e.g., expression = 0.5), even if the small-value gene is more informative for classification.

📊 What is PCA (Principal Component Analysis)?

PCA is an unsupervised dimensionality reduction technique. It finds new axes (principal components) that capture the most variance in the data.

  • PC1 — the direction of maximum spread across all 94 patients
  • PC2 — the second most spread direction, perpendicular to PC1
  • PC16 — the 16th direction, still explaining some variance

Each component is a linear combination of all 3,571 genes. You lose the ability to say "gene X is important" — instead you have "component 1, which is mostly gene A + gene B - gene C..."

PCA limitation: It doesn't know about class labels (normal vs tumor). The direction of maximum variance might be completely unrelated to cancer classification. That's why PCA often underperforms LDA for supervised tasks.

🎯 What is LDA (Linear Discriminant Analysis)?

LDA is a supervised technique. Unlike PCA which maximizes variance, LDA maximizes the ratio of between-class variance to within-class variance. In simple terms: it finds the direction where the two groups are most separated.

For binary classification (2 classes), LDA produces exactly 1 discriminant axis. All 94 patients are projected onto this single number, and a threshold separates normal from tumor.

# LDA objective (simplified): # maximize: (mean_tumor - mean_normal)² # ───────────────────────────── # var_tumor + var_normal # Fisher's Linear Discriminant criterion
Why LDA transfers to Golub: The gene direction that separates normal from tumor in PONE is related to general cancer biology. When applied to Golub (ALL vs AML), this direction still partially separates the two leukemia types — they share common cancer gene expression patterns.

🤖 The Five Classifiers

  • SVM (linear): Finds the hyperplane with maximum margin between classes. Excellent for high-dimensional data. The linear kernel assumes classes are linearly separable.
  • SVM (rbf): Radial Basis Function kernel — maps data to higher dimensions implicitly. Can find non-linear boundaries. Needs tuning of C and gamma.
  • KNN (k=5): Classifies by majority vote of 5 nearest neighbors. Simple but sensitive to irrelevant features (the curse of dimensionality).
  • Decision Tree: Learns a tree of yes/no splits on gene values. Interpretable but prone to overfitting on small datasets.
  • Random Forest: 100 decision trees on random subsets of data and features. Averages their predictions. Built-in feature selection via importance scores.

🔬 Feature Selection — 3 Families

Feature selection picks the most informative original genes (keeping gene names), unlike PCA/LDA which create new artificial dimensions.

  • Filter (ANOVA, MI): Score each gene independently. ANOVA tests statistical difference between groups. Mutual Information measures information shared with labels. Fast and classifier-agnostic.
  • Wrapper (RFE): Use SVM to rank genes, eliminate the weakest, repeat. Finds the best subset specifically for the chosen classifier. Slow but powerful.
  • Embedded (Lasso, RF): Selection happens inside model training. Lasso (L1) penalizes model complexity, driving unimportant gene coefficients to zero. RF importance ranks genes by Gini reduction.
Result: In the latest run, LDA reaches the top average balanced accuracy (0.9411). Selection-only methods with 94 genes are still useful, and the extraction+selection union is best on KNN (0.9900). Only a small subset of genes carries the classification signal.

🔀 Cross-Dataset Transfer — Why It's Hard

Training on PONE and testing on Golub is a domain transfer problem. Three things make this difficult:

  • Different feature spaces: 3,571 vs 7,129 genes → can't directly apply the model
  • Different labels: Normal/Tumor ≠ ALL/AML — different biological questions
  • Different populations: Different hospitals, protocols, measurement platforms

The proxy approach (using the first 3,571 Golub genes) is an approximation — in reality, you'd want to match genes by name across both datasets for a proper transfer experiment.

📏 Why Balanced Accuracy (Not Just Accuracy)?

With 47 normal and 47 tumar patients, random or one-class prediction gives around 50% accuracy. Balanced accuracy remains the safest metric when comparing methods across different setups.

Balanced Accuracy = (Sensitivity + Specificity) / 2 Sensitivity = True Positives / (TP + FN) # tumors correctly detected Specificity = True Negatives / (TN + FP) # normals correctly excluded # Dummy classifier: Sensitivity=0, Specificity=1 # Balanced Accuracy = (0 + 1) / 2 = 0.50 ← correctly shows it's useless
In medicine: False Negatives (missing a tumor) are more dangerous than False Positives (false alarm). So sensitivity is prioritized. For cancer screening: better to over-detect and confirm with biopsy than to miss a real tumor.
